# 坐标变换实现

> 日期: 2026-06-11

## 概述

实现方案 A（轻量方案）：静态地图渲染 + 手动标定注入 `map→odom` TF + 纯 LiDAR SLAM 里程计。不引入 AMCL，保持机器人零改动。

同时重构 TB4 / Go2 面板布局，将摄像头缩略图紧凑嵌入右侧侧边栏，删除冗余面板。

## 架构

```
地面站 (本机)
├── unitree_bridge_node    → odom→base TF (来自 Go2 LiDAR SLAM)
├── rosbridge_websocket     → ws://0.0.0.0:9090
└── Web 前端
    ├── StaticMapLayer      本地 PGM+YAML 静态地图（707.pgm + 707.yaml）
    ├── WorldFrameStore     标定数据 localStorage 持久化
    ├── 标定面板 UI         输入 X/Y/θ，注入 map→odom TF
    └── RobotLayer          map→odom→base 链路定位
```

## TF 树

```
map ──(标定注入, 静态)──▶ odom ──(LiDAR SLAM, 20Hz)──▶ base
```

- `odom→base`：Go2 LiDAR SLAM 里程计，平滑连续
- `map→odom`：前端标定面板注入，一次性静态变换
- RobotLayer 自动遍历 `map→odom→base` 得到 `map_T_base`

## 标定工作流

```
1. 将机器人放在地图上已知坐标点
   (如 location_707 中的"初始点": x=-3.24, y=3.1, θ≈89.3°)

2. 打开 web 界面 → 工具栏点「标定」
   → 选择机器人 → 输入 X/Y/θ
   → 点「应用标定」

3. map→odom TF 注入到客户端的 TF2JS 树
   → RobotLayer 自动获取 map_T_base
   → 机器人跳到地图正确位置

4. 数据自动保存到 localStorage
   → 下次打开页面自动恢复标定
```

## 新增文件

### `src/ros_web_gui_app/src/utils/StaticMapLoader.ts`
从 URL 加载 YAML + PGM 文件，返回 `OccupancyGrid`：
- 请求 YAML → 解析 `image`/`resolution`/`origin`/`free_thresh`/`occupied_thresh`/`negate`
- 解析 PGM（P5 二进制格式）→ 像素数据
- 按 ROS map_server 约定计算占用概率

### `src/ros_web_gui_app/src/components/layers/StaticMapLayer.tsx`
场景级静态地图渲染层，继承 `BaseLayer`：
- **不从 ROS 订阅话题**（`getMessageType()` 返回 `null`）
- `PlaneGeometry` + `DataTexture` 渲染
- 锚点设在左下角，位置由 YAML `origin` 决定
- 支持 `colorMode`、`alpha`、`height` 配置
- 通过 `load()` 异步加载，`setConfig()` 控制显隐

### `src/ros_web_gui_app/src/utils/WorldFrameStore.ts`
标定数据持久化（localStorage）：
- 每台机器人保存 `{ x, y, theta }` 偏移量
- `getOffset(robotId)` / `setOffset(robotId, offset)` / `removeOffset(robotId)`
- `onChange()` 变更监听回调
- 数据损坏时自动重置

## TF2JS 变换分析

对 `tf2js.ts` 中的 `findTransform()` 进行了仔细分析，**确认不存在实际 bug**：

- 单级纯平移变换 → ✓ 正确
- 单级纯旋转变换 → ✓ 正确（测试注释中的四元数半角误解导致标记为 "TODO"）
- 单级平+旋转变换 → ✓ 正确
- 多级 TF 链组合 (`map→odom→base`) → ✓ 正确

未修改 `tf2js.ts`。
