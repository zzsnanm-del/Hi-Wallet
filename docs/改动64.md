# 改动 64 — Go2 Web Dashboard 全面优化与 TB4 多机器人集成

## 一、话题节流（带宽优化）

**文件：** `src/go2_web_bridge/go2_web_bridge/unitree_bridge_node.py`

桥接节点新增基于时间的节流机制，为所有高频话题设置最小发布间隔（均为 ROS2 参数可动态调参）：

| 参数 | 间隔 | 频率 | 影响话题 |
|------|------|------|----------|
| `throttle.battery` | 120s | 1次/2分钟 | battery_state, battery/voltage, battery/current, battery/percent |
| `throttle.diagnostics` | 10s | 0.1 Hz | /diagnostics |
| `throttle.body_height` | 1s | 1 Hz | /robot/body_height |
| `throttle.foot_force` | 1s | 1 Hz | /robot/foot_force |
| `throttle.gait_type` | 0.5s | 2 Hz | /robot/gait_type |
| `throttle.point_cloud2` | 0.2s | 5 Hz | /point_cloud2 |
| `throttle.imu` | 0.1s | 10 Hz | /imu/data |
| `throttle.velocity` | 0.1s | 10 Hz | /robot/velocity/x, /y, /yaw |
| `throttle.joint_states` | 0.05s | 20 Hz | /joint_states |
| `throttle.odom` | 0.05s | 20 Hz | /odom |

**效果：** 总消息量从 ~1,540 msg/s 降至 ~89 msg/s，减少约 94%。

相关文档：[THROTTLE_DESIGN.md](THROTTLE_DESIGN.md)

## 二、前端速度 SMA 平滑防抖

**文件：** `src/ros_web_gui_app/src/components/panels/RobotCorePanel.tsx`

三个独立 Float32 话题（velocity X/Y/Yaw）通过 rosbridge 分开到达，中间渲染帧会闪现 `0.00` 或 `undefined`。修复：

1. **根因：** 5 处 `SportStateData` 默认值 `velocity: []` 改为 `velocity: [0, 0, 0]`，确保数组始终 3 元素
2. **显示兜底：** `${(velocity[i] ?? 0).toFixed(2)}` 替代 `velocity[i]?.toFixed(2)`
3. **SMA 平滑：** 4 样本滑动窗口（~400ms），消除分开到达导致的数据抖动
4. **Legacy 安全：** IIFE 保证任意长度输入数组输出始终 3 元素

## 五、地图订阅修复 — 去 MapManager 依赖

**问题：** 多机器人重构后 `MapManager.initialize()` 不再被调用，而 `GridLayer` 和 `OccupancyGridLayer` 在 `config.topic === '/map'` 时走 MapManager 路径（注册 listener 等数据），导致两个机器人的地图都无数据。

**修复：** 两个 layer 完全移除 MapManager 依赖，统一走直接 rosbridge 订阅。

**修改文件：**
- `src/ros_web_gui_app/src/components/layers/GridLayer.tsx` — 移除 MapManager import、`mapManager`/`handleMapUpdate` 字段、`/map` 分支判断、dispose 中的 listener 清理。构造函数始终 `if (config.topic) this.subscribe(...)`。
- `src/ros_web_gui_app/src/components/layers/OccupancyGridLayer.tsx` — 同上。`update()` 移除 `/map` 提前 return；`setConfig()` topic 变更走 `unsubscribe()`+`subscribe()`；`getMapMessage()` 直接返回 `this.lastMessage`。

**数据流（修复后）：**
```
每个机器人:
  OccupancyGridLayer.subscribe('/map')
    → rosbridge CBOR-raw 反序列化
    → renderMap() → DataTexture(RGBA) → MeshBasicMaterial
    → TF2JS(robot) 坐标变换
    → robotGroup.add(mesh)

  GridLayer.subscribe('/map')
    → update() → GridHelper(按地图尺寸)
```

**Go2 vs TB4 地图区别：** 两者话题名和类型完全一致 (`/map`，`nav_msgs/OccupancyGrid`)。每个机器人的 LayerManager 独立订阅自己连接上的 `/map`，通过各自 TF2JS 解析原点坐标。TB4 的 Group offset (1m) 确保两张地图在场景中不重叠。
