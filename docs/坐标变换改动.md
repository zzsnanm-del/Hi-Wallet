# 坐标变换改动

> 日期: 2026-06-11

## 概述

添加 AMCL（自适应蒙特卡洛定位）实现机器人在静态地图上的精准 2D 定位。不再依赖纯里程计推算，而是通过粒子滤波持续匹配激光扫描与地图，发布 `map→odom` 修正变换，构成完整的 `map→odom→base` TF 链。

支持两台机器人分别定位，web 前端提供初始位姿标定 UI。

## 架构

```
地面站 (本机)
├── map_server        → /map (静态地图, latched)
├── amcl (go2)        → /tf (map→go2/odom), /go2/amcl_pose
├── amcl (tb4)       → /tf (map→tb4/odom), /tb4/amcl_pose
├── unitree_bridge    → /tf (odom→base), /odom, /joint_states 等
├── rosbridge          → ws://0.0.0.0:9090
└── Web 前端
    ├── StaticMapLayer      静态 PGM+YAML 地图渲染
    ├── RobotLayer          map→base 机器人位姿
    ├── InitialPosePanel    初始位姿标定 UI
    └── 固定帧默认 map      全局坐标系显示
```

## TF 树（每台机器人独立）

```
map ──(AMCL 持续修正)──▶ odom ──(LiDAR SLAM 20Hz)──▶ base ──(URDF)──▶ sensors
```

- `odom→base`：由 unitree_bridge_node 发布，来自 Go2 LiDAR SLAM 里程计，平滑连续
- `map→odom`：由 AMCL 发布，粒子滤波匹配 scan↔map 后计算出的修正量，持续更新
- 完整链路 `map→base` = `map→odom` × `odom→base`

## 新增文件

### `src/go2_web_bridge/config/amcl_params_go2.yaml`
AMCL 参数文件：
- `base_frame_id: base`，`odom_frame_id: odom`，`global_frame_id: map`
- 激光模型：`likelihood_field`，最大匹配距离 2.0m
- 粒子数：min=500, max=2000
- 更新阈值：平移 0.2m / 旋转 0.2rad

### `src/go2_web_bridge/config/amcl_params_tb4.yaml`
TB4 的 AMCL 参数（如 TB4 未自带 AMCL）。

### `src/ros_web_gui_app/src/utils/StaticMapLoader.ts`
从本地 URL 加载 YAML + PGM 文件，调用 `mapImporter.parsePGMAndYAML()` 返回 `OccupancyGrid`：
- 请求 YAML 文件 → 解析 `image`/`resolution`/`origin` 等字段
- 请求 PGM 文件（二进制）→ 传给 `parsePGMAndYAML()`
- 返回完整 `OccupancyGrid`（含 `frame_id: 'map'`）

### `src/ros_web_gui_app/src/components/layers/StaticMapLayer.tsx`
静态地图渲染层，继承 `BaseLayer`：
- **不从 ROS 订阅话题**（`getMessageType()` 返回 `null`）
- 用 `THREE.PlaneGeometry` + `THREE.DataTexture` 渲染
- 地图位置由 YAML `origin` 字段决定：`mesh.position = (origin.x, origin.y, height)`
- 不应用额外 TF 变换（静态地图固定在 `map` 坐标系原点）
- 支持 `colorMode`、`alpha`、`height` 配置
- 使用 `paletteColorCached()` 做 occupancy → RGBA 映射

### `src/ros_web_gui_app/src/utils/WorldFrameStore.ts`
全局世界坐标系标定存储：
- 每台机器人保存 `{ x, y, theta }` 标定偏移
- 持久化到 `localStorage`
- 支持变更监听（`onChange` 回调）
- 提供 `getOffset(robotId)` / `setOffset(robotId, offset)` 接口

### `src/ros_web_gui_app/src/components/panels/InitialPosePanel.tsx`
初始位姿标定面板：
- 显示当前选中机器人的标定数据
- 输入框：X（m）、Y（m）、θ（弧度/度切换）
- 「发布初始位姿」按钮 → 向 AMCL 发布 `/initialpose` (geometry_msgs/PoseWithCovarianceStamped)
- 「保存」按钮 → 写入 WorldFrameStore + localStorage
- 标定时自动关闭相机跟随模式

## 修改文件

### `src/go2_web_bridge/launch/dashboard.launch.py`
- 新增 `map_server` 节点：从 `public/maps/707.yaml` 发布 `/map` 话题
- 新增 `amcl` 节点（Go2 命名空间 `/go2`）：订阅 `/scan` 和 `/map`，发布 `map→odom` TF
- 新增 `amcl` 节点（TB4 命名空间 `/tb4`）：订阅 `/tb4/scan` 和 `/map`（如 TB4 未自带）
- 如 Go2 无 LaserScan 输出，新增 `pointcloud_to_laserscan` 节点
- `unitree_bridge_node` 改为命名空间 `/go2` 下运行

### `src/go2_web_bridge/go2_web_bridge/unitree_bridge_node.py`
- `frame_id` 支持命名空间前缀（通过 ROS2 Node 的 `get_namespace()` 自动获取）
- 可选：新增 `/initialpose` 订阅，收到初始位姿后调整 `odom` 帧偏移

### `src/ros_web_gui_app/src/utils/mapImporter.ts`
- 导出 `parsePGMAndYAML()` 函数（原为模块私有）
- 新增 YAML 参数解析：`negate`、`occupied_thresh`、`free_thresh`、`mode`
- 按 ROS map_server 约定计算占用概率：`occ_prob = negate ? (1 - ratio) : ratio`
- **移除 Y 轴翻转**：PGM 行序与 THREE.js DataTexture 一致，无需 `height - 1 - y`

### `src/ros_web_gui_app/src/utils/tf2js.ts`
修复组合变换计算 bug：
- 修正 `findTransform()` 中两级 TF 叠加的矩阵运算
- `map→odom` + `odom→base` 的正确组合方式
- 对齐测试文件 `tf2js.test.ts` 中的 TODO 用例

### `src/ros_web_gui_app/src/components/panels/ViewportPanel.tsx`
- `DEFAULT_FIXED_FRAME` 从 `'odom'` 改为 `'map'`
- 场景初始化时创建 `StaticMapLayer`，异步加载 `/maps/707.yaml`
- Displays 面板新增 "Static Map" 开关（替代 ROS `/map` 话题地图项）
- 工具栏新增「标定」按钮 → 打开 InitialPosePanel
- 移除机器人硬编码偏移 `ROBOT_OFFSETS`（由 AMCL 提供真实位置）
- 每台机器人使用独立命名空间的 TF 帧

### `src/ros_web_gui_app/src/components/layers/RobotLayer.tsx`
- 有了 `map→odom` 变换后，`findTransform('map', 'base')` 直接返回全局坐标
- 不再需要 odom fallback（map 帧始终可用）
- relocalize 模式下支持发布 `/initialpose`：
  - 调用 `connection.publish('/initialpose', 'geometry_msgs/msg/PoseWithCovarianceStamped', ...)`
  - Covariance 初始设为较低置信度（位置 ±0.25m，朝向 ±20°），让 AMCL 粒子有一定散布空间
- 多机器人模式下使用各自的命名空间（`/go2/initialpose` vs `/tb4/initialpose`）

### `src/ros_web_gui_app/src/components/layers/OccupancyGridLayer.tsx`
- 保留对 ROS `/map` 话题的订阅支持（向后兼容）
- 静态地图场景下优先使用 StaticMapLayer

### `src/ros_web_gui_app/src/components/layers/GridLayer.tsx`
- 新增 `setMapDimensions()` 公开方法，静态地图加载后同步网格尺寸
- 网格从 StaticMapLayer 获取尺寸，不再依赖 MapManager 的话题数据

### `src/ros_web_gui_app/src/context/FleetContext.tsx`
- 机器人配置新增 `namespace` 字段（如 `go2`、`tb4`）
- 用于构造命名空间话题路径（如 `/go2/initialpose`）

### `src/ros_web_gui_app/src/utils/RosbridgeConnection.ts`
- 新增 `publishInitialPose(x, y, theta, namespace?)` 便捷方法
- 自动构造 `geometry_msgs/msg/PoseWithCovarianceStamped` 消息

## 标定工作流

```
1. 启动系统 ./start.sh
   └── map_server 加载 707.yaml → /map
   └── AMCL 节点启动，等待 /initialpose

2. 放置机器人
   └── 将 Go2 放到已知地图坐标点
      （如 初始点: x=-3.24, y=3.1, θ≈89.3°）

3. 打开 web 标定面板
   └── 选择机器人: Go2
   └── 输入 X / Y / θ
   └── 点击「发布初始位姿」

4. AMCL 粒子收敛
   └── Web 界面看到机器人从默认位置跳到标定位置
   └── 粒子逐步收敛，位姿趋于稳定

5. 保存标定数据
   └── 点击「保存」→ localStorage
   └── 下次开机可快速加载上次标定值

6. 对 TB4 重复步骤 2-5
```

## AMCL 关键参数

| 参数 | Go2 | TB4 | 说明 |
|------|-----|-----|------|
| `min_particles` | 500 | 500 | 最少粒子数 |
| `max_particles` | 2000 | 2000 | 最多粒子数 |
| `laser_model_type` | likelihood_field | likelihood_field | 似然场，计算快 |
| `laser_likelihood_max_dist` | 2.0 | 2.0 | 激光最大匹配距离(m) |
| `update_min_d` | 0.2 | 0.2 | 平移阈值(m) |
| `update_min_a` | 0.2 | 0.2 | 旋转阈值(rad) |
| `recovery_alpha_slow` | 0.001 | 0.001 | 慢恢复权重 |
| `recovery_alpha_fast` | 0.1 | 0.1 | 快恢复权重 |
| `initial_pose.x/y` | 0.0 | 0.0 | 默认初始位置（等标定覆盖） |
| `initial_cov_xx/yy` | 0.25 | 0.25 | 初始位置方差 |
| `initial_cov_aa` | 0.0685 | 0.0685 | 初始朝向方差(~20°) |

## 风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|------|---------|
| Go2 不发布 LaserScan | 中 | 加 `pointcloud_to_laserscan` 节点从 `/utlidar/cloud_deskewed` 生成 `/scan` |
| LiDAR SLAM 里程计跳变导致 AMCL 发散 | 低 | AMCL recovery_alpha 参数可吸收偶尔跳变 |
| 两台机器人 topic/TF 冲突 | 高 | ROS2 namespace 隔离（`/go2/`, `/tb4/`） |
| TF2JS 两级变换计算仍有 bug | 中 | 先跑通 tf2js.test.ts 所有用例 |
| 地图 YAML origin 与物理世界不对齐 | 中 | 标定面板支持 offset 微调，或重新测量 origin |
| DDS 发现跨机器不稳定 | 低 | 局域网环境下 ROS2 Humble DDS 已验证可行 |
| TB4 已有自带 AMCL，重复启动冲突 | 中 | 检测 TB4 是否已有 `/amcl_pose` 话题，有则跳过本地 AMCL |
