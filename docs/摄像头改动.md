# 摄像头改动

> 日期: 2026-06-08

## Tb4 视频流地址更新

Tb4 的视频流从机器人本机 IP 迁移到了独立摄像头服务器 `<camera-ip>`。

### 修改文件

[DashboardPage.tsx](../src/ros_web_gui_app/src/components/pages/DashboardPage.tsx#L142)

| 项目 | 旧 | 新 |
|------|-----|-----|
| IP | `${selectedRobot.ip}` (动态取 Tb4 IP) | `<camera-ip>` (固定摄像头服务器) |
| 路径 | `/stream` | `/stream`（不变） |
| topic | `/oakd/rgb/image_raw` | 不变 |
| type | `mjpeg` | 不变 |

### 完整 URL

```
http://<camera-ip>:7654/stream?topic=/oakd/rgb/image_raw&type=mjpeg
```

### 注意事项

- `/stream_viewer` 返回 HTML 查看器页面（`text/html`），不能在 `<img>` 标签中直接使用
- `/stream` 返回原始 MJPEG 流（`multipart/x-mixed-replace`），有 CORS 头 `Access-Control-Allow-Origin: *`
- CameraVideoPanel 用 `<img>` 标签加载 MJPEG 流，所以必须用 `/stream` 端点

## Go2 摄像头/视频面板

### 新增视频流

Go2 设备状态面板新增 CameraVideoPanel，显示 RealSense 深度相机的 RGB 彩色画面。

### 修改文件

- [CameraVideoPanel.tsx](../src/ros_web_gui_app/src/components/panels/CameraVideoPanel.tsx) — 重写
- [DashboardPage.tsx](../src/ros_web_gui_app/src/components/pages/DashboardPage.tsx#L145) — Go2 分支添加视频面板
- [App.css](../src/ros_web_gui_app/src/App.css) — 新增 `.cam-thumb` / `.cam-lightbox` 样式

### DashboardPage 改动

| 项目 | 旧 | 新 |
|------|-----|-----|
| Go2 设备面板 | 仅 grid-3 (6 个面板) | grid-3 第一项为 CameraVideoPanel |
| videoSrc | 无 | `http://<go2-ip>:7654/stream?topic=/camera/color/image_raw&type=mjpeg` |

### CameraVideoPanel 重写

| 项目 | 旧 | 新 |
|------|-----|-----|
| 话题状态指标 | 彩色图像/深度图像/前端视频流/VideoHub 共 5 行 | 仅保留帧率 1 行 |
| 视频尺寸 | `minHeight: 280px`，独立全宽 | grid-3 内自适应，与其他面板等大 |
| 点击放大 | 无 | 点击缩略图弹出全屏 lightbox，Esc/点击背景关闭 |

### 完整 URL

```
http://<go2-ip>:7654/stream?topic=/camera/color/image_raw&type=mjpeg
```

### 样式说明

- `.cam-thumb` — 16:9 缩略图容器，hover 蓝色边框 + "🔍 点击放大" 提示
- `.cam-lightbox` — 全屏 92% 不透明度遮罩，fadeIn 动画
- `.cam-lightbox-close` — 右上角圆形关闭按钮
- 点击打开不依赖 `onLoad`（MJPEG 流触发不可靠），仅判断 `videoSrc` 是否存在

## 缩略图黑屏修复

### 问题

小屏/缩略图模式下 MJPEG 视频流首次连接容易失败，`onError` 触发后将图片 `display: none`，仅剩 `.cam-thumb` 容器的黑色背景（`background: #000`），表现为黑屏。但点击打开 lightbox 时创建了新的 `<img>` 元素，重新连接往往成功，所以"点进去才能有画面"。

### 修改文件

- [CameraVideoPanel.tsx](../src/ros_web_gui_app/src/components/panels/CameraVideoPanel.tsx) — 修复缩略图错误处理逻辑
- [App.css](../src/ros_web_gui_app/src/App.css) — 新增错误占位样式

### CameraVideoPanel 改动

| 项目 | 旧 | 新 |
|------|-----|-----|
| 加载失败处理 | `display: none` 隐藏图片 → 黑屏 | 显示错误占位 UI，支持点击重试 |
| 重试机制 | 无（只能刷新页面） | `retryKey` state 递增 → 强制 `<img>` 重新挂载 |
| 打开 lightbox | 直接打开，忽略缩略图错误状态 | 先重置错误并重试，再打开 lightbox |

### 新增样式

- `.cam-thumb-error` — 错误占位容器，居中 flex 布局，黑色背景
- `.cam-thumb-error-icon` — 📷 图标
- `.cam-thumb-error-text` — "视频流加载失败"
- `.cam-thumb-error-hint` — accent 色 "点击重试"
