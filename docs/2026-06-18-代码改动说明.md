# 2026-06-18 具体代码改动说明

## 一、改动目标

本次改动围绕三个目标：

1. 修复 TB4 OpenClaw 请求被错误代理到 Go2 的问题。
2. 修复刷新页面后 AI 助手“顶部机器人状态”和“当前聊天记录”不一致的问题。
3. 将 OpenClaw token、机器人地址、摄像头地址统一迁移到 `.env` 管理，并在 README 中说明私有阶段和开源阶段的处理策略。

## 二、新增文件

### `.env`

用途：私有开发阶段的真实环境配置。

包含内容：

- Go2 OpenClaw 网关地址。
- TB4 OpenClaw 网关地址。
- Go2/TB4 OpenClaw token。
- 浏览器端私有开发可见的 Go2/TB4 token。
- Go2/TB4 摄像头流地址。

说明：

- 当前仓库是私有仓库，所以 `.env` 按需求保留真实值。
- 开源前必须删除真实值、轮换 token，并处理 git 历史。

### `.env.example`

用途：提供环境变量模板，方便其他开发者了解需要配置哪些值。

该文件只放占位符，不放真实 token。

## 三、配置和代理改动

### `src/ros_web_gui_app/vite.config.ts`

主要改动：

- 使用 `loadEnv(mode, repoRoot, '')` 从仓库根目录读取 `.env`。
- 设置 `envDir: repoRoot`，让前端也能读取根目录下的 `VITE_*` 变量。
- 新增 OpenClaw 代理目标：
  - `OPENCLAW_GO2_URL`
  - `OPENCLAW_TB4_URL`
- 新增代理请求头注入：
  - `OPENCLAW_GO2_TOKEN`
  - `OPENCLAW_TB4_TOKEN`
- 调整 proxy 顺序，把 `/api/ai-tb4` 放到 `/api/ai` 前面。

解决的问题：

- 避免 `/api/ai-tb4` 被 `/api/ai` 提前匹配。
- 避免 TB4 请求错误转发到 Go2。
- 让 token 不再硬编码在 Vite 配置中。

### `serve.py`

主要改动：

- 新增简单 `.env` 读取函数。
- 启动时读取仓库根目录 `.env`。
- 使用以下变量配置代理：
  - `OPENCLAW_GO2_URL`
  - `OPENCLAW_TB4_URL`
  - `OPENCLAW_GO2_TOKEN`
  - `OPENCLAW_TB4_TOKEN`
- 如果浏览器请求没有带 `Authorization`，由 `serve.py` 自动注入 token。
- 保留旧变量兼容：
  - `AI_PROXY_GO2`
  - `AI_PROXY_TB4`
  - `AI_PROXY_TARGET`

解决的问题：

- Python 静态服务模式下也能使用统一 `.env` 配置。
- OpenClaw token 不再散落在源码中。

## 四、AI 助手状态同步改动

### `src/ros_web_gui_app/src/utils/openclawApi.ts`

主要改动：

- 新增常量：
  - `OPENCLAW_GO2_BASE_URL`
  - `OPENCLAW_TB4_BASE_URL`
- 新增函数：
  - `getOpenClawTokenForBaseUrl(baseUrl)`
- 默认设置中的 token 改为从 `VITE_OPENCLAW_GO2_TOKEN` 读取。
- 请求头逻辑调整为：只有 token 非空时才加 `Authorization`。

解决的问题：

- 私有开发阶段 Web 设置页可以显示 token。
- 开源阶段删除 `VITE_OPENCLAW_*_TOKEN` 后，前端 token 会自然为空。
- 同时保留代理注入 token 的能力。

### `src/ros_web_gui_app/src/hooks/useOpenClawChat.ts`

主要改动：

- 新增 `ai_chat_active_conversation_id`，持久化当前激活对话。
- 初始化时根据激活对话的 `baseUrl` 同步 settings。
- 新建对话时写入当前机器人 `baseUrl`。
- 切换对话时同步 settings 到该对话对应机器人。
- 删除当前对话后，优先切到同机器人下的其他对话。
- 历史对话没有 `baseUrl` 时默认归为 Go2。
- settings 切到 Go2/TB4 时，自动回填对应 `VITE_OPENCLAW_*_TOKEN`。

解决的问题：

- 刷新后不会再出现“聊天内容是 TB4，但顶部显示 Go2”的错位。
- 切换不同机器人的历史对话时，顶部状态、输入框、助手标签会同步变化。

### `src/ros_web_gui_app/src/components/ai/AIChatDialog.tsx`

主要改动：

- 使用 `OPENCLAW_GO2_BASE_URL` 和 `OPENCLAW_TB4_BASE_URL` 替代字符串硬编码。
- 顶部 GO2/TB4 切换时，根据目标机器人回填 token。
- 空对话提示从固定 Go2 改为当前机器人名称。
- 输入框 placeholder 从固定 Go2 改为当前机器人名称。
- 给 `AIChatMessage` 传入动态 `assistantLabel`。

解决的问题：

- UI 文案和当前机器人保持一致。
- 私有开发阶段切换机器人后，设置页 token 能显示对应机器人的值。

### `src/ros_web_gui_app/src/components/ai/AIChatMessage.tsx`

主要改动：

- 新增 `assistantLabel` 参数。
- 助手消息标签从固定 `GO2 >` 改为动态显示。

解决的问题：

- TB4 对话中助手消息不再错误显示为 `GO2 >`。

### `src/ros_web_gui_app/src/components/ai/AIChatSettings.tsx`

主要改动：

- 快捷配置使用 `OPENCLAW_GO2_BASE_URL` 和 `OPENCLAW_TB4_BASE_URL`。
- 切换 Go2/TB4 preset 时，token 从 `getOpenClawTokenForBaseUrl()` 回填。
- 拉取模型列表时使用当前 preset 对应 token。

解决的问题：

- 打开设置页可以看到当前机器人 token。
- 切换 preset 后 token 不再被清空。

## 五、摄像头配置改动

### `src/ros_web_gui_app/src/components/pages/DashboardPage.tsx`

主要改动：

- Go2 摄像头地址从硬编码改为 `import.meta.env.VITE_GO2_CAMERA_URL`。
- TB4 摄像头地址从硬编码改为 `import.meta.env.VITE_TB4_CAMERA_URL`。

解决的问题：

- 摄像头地址统一放到 `.env` 管理。
- 后续更换机器人 IP 时不需要改源码。
